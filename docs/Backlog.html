Class: Backlog

Backlog(id, title, descriptionopt, ratesopt, finalRateopt, idopt)

Classe à utiliser pour créer une fonctionnalité

Constructor

new Backlog(id, title, descriptionopt, ratesopt, finalRateopt, idopt)

Si l'id est défini avec undefined, alors on instancie un objet Backlog à partir des paramètres title, description et rate, sinon on instancie en récupérant les données enregistrer à partir de l'id
Parameters:

id string Id de la fonctionnalité

 Description (facultatif)
rates Array.<RateObject> <optional>
 Notes attribuées à la fonctionnalité de 0 à 100, et -1 est l'état pour non noté, chaque note est liée à l'id d'un User (facultatif)
finalRate number <optional>
 Note finale de la fonctionnalité, de 0 à 100, -1 pour en cours de notation, -2 pour non noté

 Donner l'id seulement si l'on définit une fonctionnalité déjà existante
